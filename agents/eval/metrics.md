# 記事品質 評価基準（metrics.md）

> Eval Agentが記事を採点する際の基準。各項目1-10のスケールで評価する。

## 評価項目

### 1. 情報の正確性 (1-10)
- **10**: すべての事実にソースURLが付いており、内容が正確
- **7**: 大部分にソースがあり、明らかな誤りがない
- **5**: ソースが一部欠けている、または軽微な不正確さがある
- **3**: ソースが不十分、または事実誤認がある
- **1**: ソースなし、または重大な事実誤認がある

### 2. 可読性 (1-10)
- **10**: 中学生でも理解でき、文が短く、構成が明確
- **7**: 一般読者が問題なく読める
- **5**: 一部に専門用語の説明がない、または文が長い
- **3**: 読みにくい部分が多い
- **1**: 専門家でないと理解できない

### 3. 情報密度 (1-10)
- **10**: 無駄な文がなく、すべての文がユニークな情報を提供
- **7**: ほぼ無駄がなく、情報量が適切
- **5**: やや冗長な部分がある
- **3**: 水増し感がある、同じことの繰り返し
- **1**: 内容が薄く、ほぼ情報がない

### 4. SEO (1-10)
- **10**: タイトル・メタデータ・見出し構造がすべて最適化されている
- **7**: 主要なSEO要素が揃っている
- **5**: 一部のSEO要素が欠けている
- **3**: SEO対策がほぼされていない
- **1**: タイトルやメタデータが未設定

### 5. 独自性 (1-10)
- **10**: 独自の分析・解説・視点が含まれ、他にない価値がある
- **7**: 一定の分析や解説が含まれている
- **5**: ソースの要約が中心だが、多少の独自視点がある
- **3**: ほぼソースのコピペに近い
- **1**: 完全なコピペ

### 6. 実用性 (1-10)
- **10**: 読者がすぐに行動に移せる具体的な情報がある
- **7**: 参考になる実用的な情報が含まれている
- **5**: 知識として有用だが、すぐに使える情報は少ない
- **3**: 情報が一般的すぎて実用性が低い
- **1**: 読者にとって行動可能な情報がない

### 7. 出典数 (1-10)
- **10**: 5つ以上の出典リンクがあり、すべて一次ソース
- **7**: 3つ以上の出典リンクがあり、半数以上が一次ソース
- **5**: 3つの出典リンクがある（最低基準達成）
- **3**: 1-2つの出典リンクしかない
- **1**: 出典リンクなし

### 8. 一次ソース比率 (1-10)
- **10**: 出典の100%が一次ソース（公式発表、プレスリリース、論文）
- **7**: 出典の50%以上が一次ソース
- **5**: 出典の30-50%が一次ソース
- **3**: 出典のほぼすべてが二次ソース（メディア報道のみ）
- **1**: 出典の信頼性が不明、または出典なし

## 総合スコア
- 上記8項目の平均値（小数点第1位まで）
- **目標**: 初月6.0以上、3ヶ月後7.0以上、6ヶ月後8.0以上
