### 追記提案1: 冒頭ルールの強化

```markdown
## 冒頭の書き方（最重要ルール）

冒頭3行は記事で最も重要。以下のルールを厳守する。

### 禁止事項
- タイトルと同じ内容を繰り返す（読者はタイトルを読んで来ている）
- 「〜が進化しています」「〜が注目されています」等の抽象的な書き出し
- 「2026年は〜の年です」等のAI定型表現

### 推奨パターン（いずれか1つを使う）
1. **ギャップ提示**: 「導入率57%、でも現場で使っているのは5%未満」
2. **具体的な数字で衝撃**: 「3ヶ月で3400億円——AIロボットに投資マネーが殺到している」
3. **問いかけ**: 「あなたの会社でAIを実際に使っている社員は何%いますか？」
4. **反常識**: 「AIエージェント導入の95%は失敗に終わっている」
```

### 追記提案2: 比較記事のルール追加

```markdown
## 比較記事のルール

### 構成
- 全項目を同じテンプレートで書かない。項目ごとに書き出しを変える
- 各項目に必ず「弱点」「向いていない場面」を記載する
- 比較表と用途別まとめは必須（読者が最も価値を感じるパーツ）

### タイトル
- 煽りワード（「最新」「徹底比較」「これだ」「必見」）は最大1つまで
- 数字 + 具体名を入れる（例: 「AI画像生成ツール7つを用途別に比較【2026年3月版】」）

### 弱点の書き方
- 各ツールに最低1つ、具体的なデメリットを記載する
- 「〜が課題です」ではなく「〜の場面では使いにくい。理由は〜」と状況+理由で書く
- コスト感（月額の目安・1回あたりのコスト）を可能な限り明示する
```

### 追記提案3: 出典ルールの厳格化

```markdown
## 出典ルール（厳格化 2026-03-28）

### 必須ルール
1. 数字を使う場合、一次ソース（調査会社の公式レポート、企業の公式発表、政府統計）を優先する
2. 二次ソース（ブログ記事、ニュースまとめサイト）から数字を引用する場合、元の一次ソースを探して確認する
3. 一次ソースで確認できない数字には必ず[未確認]タグを付ける
4. 市場規模の数字は「何を含む市場定義か」「いつ時点の予測か」「どの調査会社のデータか」を明記する
5. 出典の利害関係（調査元が販売側である場合など）を読者にわかるよう明示する

### 禁止事項
- medium.com、greyjournal.net 等のドメインだけのリンク（記事URLを完全に指定する）
- 「著作権フリー」「無料（オープンソース）」等の法的条件を不正確に記述すること
- 複数の調査会社で数字が異なる場合に、最も大きい数字だけを採用すること
```
