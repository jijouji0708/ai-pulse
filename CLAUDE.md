# AI Pulse — AIエージェント自律運営メディア + AI開発受託

## プロジェクト概要

7体のAIエージェントが自律的に運営する事業体。2本柱:
1. **AI Pulse ブログ**: AI/テクノロジーの日本語ニュースメディア（広告収益 + ブランディング）
2. **AI開発受託**: ブログで獲得した信頼をもとに企業向けAIツール開発を受注

## 事業構造

```
ブログ（毎日自動更新） → 読者増 → 広告収益（副収入）
                       → X投稿で拡散 → 信頼構築
                       → 企業からの問い合わせ → 案件受注（本命収益）
```

## アーキテクチャ
- **ブログ**: Astro (静的サイトジェネレーター) + GitHub Pages
- **NL配信**: Buttondown (無料枠)
- **エージェント基盤**: Claude Code Scheduled Tasks
- **エージェント間通信**: ファイルシステムベース (`agents/` ディレクトリ)
- **バージョン管理**: Git

## 記事運用ルール（2026-03-27 全員会議で決定）

### 投稿数制限（AI Slop判定回避・厳守）
- **1日の新規記事公開は最大2本まで**
- 3本以上を一括公開しない（GoogleにAIコンテンツファーム判定されるリスク）
- 下書きのストックは制限なし。公開タイミングを分散する
- 記事の質を量より優先する。出典リンクのない記事は公開しない

### 1本目: ニュースまとめ（平日毎日）
- タイトル形式: 「【M/DD】AIニュース○選 — [メイントピック]」
- 3〜5件のニュースを取り上げる
- 各ニュース: 見出し + 3行解説 + 出典リンク
- 重要度「高」のニュースにはソウタ監修の技術コメントを付与
- リサーチソース: Hacker News、arXiv、TechCrunch、主要AI企業ブログ、X

### 2本目: 自由テーマ（1本目と内容が被らないこと）
- テーマ選定: CEO戦略指定70% / 自律選定30%
- 不足カテゴリを優先: 入門・用語解説、メタコンテンツ
- ユキの品質フィルタ: 独自性・具体的アクション・出典の3条件を満たすこと
- 既存記事とのポートフォリオバランスを考慮

### 品質管理プロセス
- **日次（公開前）**: ソウタ（ファクトチェック）→ ユキ（品質チェック）
- **週次**: CEO週次レビューに統合
- レビューで問題があれば修正してから公開

## 安全ルール（全エージェント共通・厳守）

1. **外部への送信・投稿は禁止**。下書き生成まで。実際の投稿は人間が承認
2. **ファイル削除禁止**。新規作成と上書きのみ
3. **外部APIへの書き込み禁止**。読み取りのみ
4. **ループ処理には最大回数を設定**。無限ループ禁止
5. **エラー時は無視して次へ進む**。ログに記録して止まらない
6. **全記事にAI生成であることを明記**
7. **不確かな情報には[未確認]タグを付ける**
8. **Git追跡**: 全変更をGitでコミット

## ファイルベース通信規約

- 各エージェントは自分のフォルダにのみ書き込む
- 他エージェントのフォルダは読み取り専用
- 日付ベースのファイル名: YYYY-MM-DD-*.md
- shared/communication/ にエージェント間メッセージを置く

## 自己改善サイクル（Eval Agent）

毎週のサイクル:
1. 過去1週間の記事品質を評価（eval/metrics.md の基準に基づく）
2. 高パフォーマンス記事 vs 低パフォーマンス記事を比較分析
3. content/style-guide.md を改善（改善版で3記事テスト → eval → 良ければ保持）
4. 改善履歴を eval/history/ に記録
5. CEO Agentに週次改善レポートを提出

## 品質基準

- E-E-A-T準拠（経験・専門性・権威性・信頼性）
- 全ての事実主張に出典リンクを付ける
- 可読性: 中学生が読んで理解できるレベル
- AI生成であることを全記事フッターに明記
